2. Why This Blog Is Designed The Way It Is

This philosophy blog was intended to be structured similarly to the Drudge Report and PhilosophyBasics.com, but with better and more modern HTML/CSS code.

The home page of this blog categorizes all my links and posts by category, with short descriptions, so that people who come to this site for the first time will be more likely to read the most important information first. On the other hand, people who read this blog on a regular basis every time a post comes out will presumably have already read most or all of this blog content, and thus they would rather navigate the blog in reverse chronological order, like how most blogs on the Internet are organized, hence why I have a separate page listing all my blog posts and links in the order that they were added to the blog.

One hope of mine is that the blog’s homepage could evolve to be a substitute of sorts for what Youtube used to be like, a place for connecting lots of different niche content creators by links that are all in one place. I didn’t start using Youtube a lot until 2016, so unfortunately, I actually never got to experience what Youtube used to be like before 2014, but everybody has said that it used to be so much better than it is now.

3. How I Created All The HTML And CSS Pages

  1. I have been typing down my philosophical beliefs for the last 4 years prior to creating this blog, all in plain text.
  2. I started off using Vim, but a year later, I switched to Emacs, and as such, I learned how to use Org-Mode, and I started converting all my old philosophy .txt files into .org mode files.
  3. Once my files were written in Org-Mode Markup Language, I could use Emacs to export the .org files into .html files, and those are what get displayed on this blog.
  4. I started off using this template that I found at joshcanhelp.com, and I customized it with my own style and formatting, and replaced the old HTML4 code with HTML5 code instead: JoshCanHelp Drudge Report HTML Template.
  5. I started creating the CSS pages from the code embedded inside the .html template that I found, and learning CSS from Wikibooks at this link: Wikibooks: Cascading Style Sheets.
  6. I finished the copy-editing on all my .org files to make the HTML output of what I had written readable and presentable.
  7. I chose github.io to host this site, since it’s capable of hosting raw .html pages.
  8. I learned how to use Git.
  9. I published this site.

However, I’m not finished designing this website yet. Ideally, I would like to display an identical header and footer on each page, and make modifications to the website and webpages to make everything more navigable, but I will probably have to learn how to write some PHP code before I can make that happen.

4. Blithering Genius’s Thoughts On What Would Make An Ideal Discussion/Blogging Platform

The ideal discussion/blogging space would have the following features:

  • Users have their own channels/blogs. (Not topic-based forum.)
  • Allow different formats: text, video, audio, image, short text.
  • Nominal and substantive freedom of speech, including anonymity.
  • Persistence of content. (No deletions by users, only deletions by mods.)
  • Disqus style comments. No user-blocking or filtering of comments.
  • Open record of likes, subs, comments to encourage propagation.
  • Ease of use, stable format.
